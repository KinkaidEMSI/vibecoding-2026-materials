Core Philosophy
🧭 What Vibe Coding Means
  • Students direct AI, not write code. The skill being taught is prompting, iterating, and thinking — not syntax memorization.
  • Success = shipped projects. A student who ships something imperfect has succeeded. A student with perfect understanding and nothing deployed has not.
  • Commits are the participation grade. At least one meaningful commit per session is the bar. Check GitHub, not code quality.
  • Idea tickets gate building. Approve them before Day 3 (Project 1) and Day 9 (Project 2). This prevents scope creep and wasted sessions.
Pre-Course Checklist
  • ✅ GitHub Pages enabled on vibecoding-2026-materials repo (Settings → Pages → main → /root)
  • ✅ Two GitHub Classroom assignments created: vibeproject1-2026- and vibeproject2-2026-
  • ✅ GEMINI_API_KEY org-level Codespaces secret set in KinkaidEMSI org settings
  • ✅ Test run: accept your own assignment in incognito, open Codespace, verify echo $GEMINI_API_KEY returns a value
  • ✅ Send GitHub account creation instructions to students the night before Day 1
  • ✅ Confirm school WiFi does not block WebSockets (critical for Codespaces) — test on school MacBook on school network
  • ✅ Have the GitHub Classroom assignment links ready to share on Day 1 (projector + Google Classroom)
⚠️ If WiFi blocks WebSockets

Codespaces will not work at all. This is the single highest risk to the course. Test it on the actual school network before Day 1. If blocked, escalate to IT — the fix is a firewall rule, not a workaround.

📋 Participation Grading Philosophy
  • Ships + commits = full credit. You are not grading code quality, design, or complexity.
  • The bar is deployment, not perfection. A broken quiz that's live on Pages and has 3 commits passes. Beautiful code that was never pushed fails.
  • Commit messages tell the story. "update" is a yellow flag. "Day 4: added interests section with three cards" shows engagement.
  • Struggling students: The minimum for full credit is a page with any content, committed, deployed. Be generous about what "done" looks like.
  • Advanced students: Encourage them to add stretch goals. Their grade doesn't change — but the demo is more impressive.
Common Issues & Fixes
IssueFirst thing to tryIf that doesn't work
Gemini not responding / no API key Terminal: echo $GEMINI_API_KEY — should show a value F1 → "Codespaces: Rebuild Container". If still blank, check org secret is scoped to the repo in KinkaidEMSI settings.
Codespace won't load / stuck spinning Hard refresh (Ctrl+Shift+R). Close tab and reopen Codespace from GitHub repo. Delete the Codespace and create a new one from the repo. Data is in git — nothing lost.
Live Server not working Check Ports panel — port 3000 should be forwarded. Right-click → Open in Browser. Right-click index.html → Open with Live Server.
git push fails / permission denied Check they're pushing the right repo. git remote -v should show their repo URL. Student may not be logged into GitHub in the Codespace. Run gh auth status.
GitHub Pages not updating Check Actions tab in repo — Pages deploy takes 1–2 min. Wait and refresh. Check Settings → Pages — confirm source is set to branch main, root /.
Student out of Codespace hours Hours reset monthly. Check github.com/settings/billing — may have hours remaining. Student uses local VS Code with the repo cloned. Git workflow is identical.
AI gave code that broke everything Check git log — if they committed before the bad paste, run git checkout HEAD -- index.html Have them re-run the prompt with explicit instruction: "Do not change [specific thing]." Remind them of the complete-file rule.
🚨 School WiFi blocking WebSockets

Codespaces uses WebSockets. If blocked, Codespaces will not work at all — the editor will fail to connect. Signs: spinning forever, "Connection to server was lost" error. Fix: IT needs to whitelist WebSocket traffic to *.github.dev and *.vscode.dev. Test this on the school network before Day 1.

Stack Summary
  • GitHub Org: KinkaidEMSI — all student repos live here
  • Student IDE: GitHub Codespaces (browser VS Code, devcontainer.json controls the environment)
  • AI Assistant: Gemini Code Assist VS Code extension + Google AI Studio (aistudio.google.com/prompts/new_chat) as backup
  • API Key: GEMINI_API_KEY stored as org-level Codespaces secret — injected automatically, never committed
  • Deployment: GitHub Pages — auto-deploys on push to main branch
  • LMS: Google Classroom for logistics, due dates, URL submission
  • Assignment tracking: GitHub Classroom — commit history is the participation record
Google AI Studio — Backup AI
🤖 When Gemini Code Assist is down, use AI Studio
  • URL: aistudio.google.com/prompts/new_chat
  • Model: Gemini 2.5 Flash (non-preview)
  • Thinking: Minimal | Grounding/Search: OFF
  • Free tier: 1,500 requests/day per account
  • Students log in with their personal Google account
  • Response time: ~17 seconds (slower than in-editor, still usable)
  • Do NOT use aistudio.google.com/app (App Builder) — 200s+ latency, unusable
