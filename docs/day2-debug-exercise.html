Debug Exercise

Every student has the same broken file. Find all 8 bugs, fix them, commit, and push. Your Pages URL goes live for the first time today.

🎯 What You're Learning

Debugging is reading carefully. Most HTML bugs are typos, missing tags, or wrong punctuation. Today you'll fix a broken file and commit the fix — the same loop you'll use for every project. By the end of class, your GitHub Pages URL will be live.

The 8 Bugs — Find Them All
BUG 1Missing closing tagEasy

An HTML element is opened but never closed. The browser will try to guess — and guess wrong.

Look at the <title> element in <head>
BUG 2Typo in attributeEasy

An attribute name is misspelled. The feature it controls will silently not work.

Check the <meta charset> tag spelling
BUG 3Wrong quote typeMedium

Someone used curly quotes ("smart quotes") instead of straight quotes. HTML won't recognize the attribute value.

Look in the <link> href attribute
BUG 4Unclosed attributeMedium

A quote is opened but never closed. Everything after it becomes part of the attribute value.

Check the <img> alt attribute
BUG 5Wrong nesting orderMedium

An element closes before its parent, or elements overlap. Nesting must be clean: open inner first, close inner first.

Look at the <strong> inside <em> in the paragraph
BUG 6Missing required attributeMedium

An element is missing an attribute that is required for it to work properly. Accessibility and function both break.

Check the <img> tag — what's always required?
BUG 7Broken link syntaxHard

A link exists but clicking it does nothing useful. The href value is wrong in a subtle way.

Look at the <a> href — is the URL structure correct?
BUG 8Invalid elementHard

A tag name doesn't exist in HTML. Browsers ignore unknown elements and their content disappears.

Scan all tag names — one is invented
The Broken File
broken-index.html — copy this into your index.html, then fix all bugs 8 BUGS
<!DOCTYPE html>
<html lang="en">
<head>
  <meta cahrset="UTF-8"> <!-- BUG 2 -->
  <title>My First Page <!-- BUG 1: missing closing tag -->
  <link rel="stylesheet" href="style.css> <!-- BUG 3: wrong quotes? BUG 4: unclosed quote -->
</head>
<body>

  <h1>Welcome to My Site</h1>

  <p>This is <em>really <strong>important</em></strong> stuff.</p> <!-- BUG 5 -->

  <img src="photo.jpg" alt="A cool photo> <!-- BUG 4 -->

  <img src="logo.png"> <!-- BUG 6: what's missing? -->

  <a href="www.google.com">Visit Google</a> <!-- BUG 7 -->

  <marquee>This is moving text!</marquee> <!-- BUG 8: not valid HTML5 -->

</body>
</html>
How to Work Through This
  • 1Copy the broken code above into your index.html (select all, paste)
  • 2Save and check Live Server. Notice what's broken visually.
  • 3Work through the bug list top to bottom. Use the hints — but figure out the fix yourself.
  • 4After each fix, save and check Live Server again. Did it help?
  • 5If you're stuck on a bug, ask Gemini: "Here is my HTML. Bug 3 says there are wrong quotes on the link href — what does that mean and how do I fix it?"
  • 6When all 8 bugs are fixed, your page should render cleanly with no visible errors.
⚠️ Don't just ask Gemini "fix my code"

You'll learn nothing. Ask about one bug at a time. Explain what you think the bug is first, then ask Gemini if you're right. That's how debugging actually works.

When you're done — commit and push

This pushes to GitHub and triggers your Pages URL to go live for the first time.

git add .
git commit -m "Day 2: fixed all 8 bugs in debug exercise"
git push
