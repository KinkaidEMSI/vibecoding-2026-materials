Project Tickets

Three projects. Build them with AI. Ship them to the internet.

How every project works: You tell AI what to build → AI writes the code → you paste it into index.html → see it live → ask for changes → git push. When it’s done, submit your GitHub Pages URL to Google Classroom.
PROJECT 01 Personal Page

Days 1–3 · Jun 8–10 · 60 min sessions

A page about you. Who are you? What do you care about? What makes you interesting? This is your corner of the internet.

STARTING PROMPT

“Build me a personal webpage. My name is [name]. I go to [school]. Things I care about: [list 3–5 things]. My aesthetic is [describe: dark and minimal / bright and colorful / retro / futuristic / chaotic / clean]. Include a short bio, a section about my interests, and a way to reach me. Make it look like something I’d actually want to show people.”

YOUR PAGE SHOULD HAVE

  • Your name and a short bio
  • At least 3 things you’re into
  • A visual style that feels like you — colors, fonts, layout
  • At least one image or icon

TO SUBMIT

Push your code → enable GitHub Pages (Settings → Pages) → copy your live URL → submit it in Google Classroom.

PROJECT 02 Interactive Page

Days 4–8 · Jun 11–18 · 60 min sessions

A page that does something. When the user clicks, types, or interacts, something changes. Pick an idea below or bring your own.

PICK AN IDEA

QuizMultiple choice on a topic you love. Show the score at the end.
Clicker gameButton that increments a counter. Add upgrades, achievements, prestige.
Random generatorClick a button, get a random result. Band name, dinner idea, dare, compliment.
Interactive storyMake choices, see different outcomes. A branching narrative.
Flashcard appClick to flip. Study vocab, history dates, anything.
Personality quizAnswer questions, get a result. “What [thing] are you?”

STARTING PROMPT

“Build me a [quiz / clicker game / random generator / interactive story / etc.] as a single HTML file. The theme is [describe your topic]. The user [describe the interaction — what they click/type and what happens]. Use a [dark / colorful / minimal] design. Everything in one HTML file with CSS and JavaScript included.”

YOUR PROJECT SHOULD HAVE

  • Something the user can click, type, or interact with
  • A visible response when they interact — something changes on screen
  • A clear purpose — someone should understand it in 5 seconds
  • A design that matches the vibe of the project

TO SUBMIT

Push your code → GitHub Pages auto-updates → copy your live URL → submit in Google Classroom.

PROJECT 03 Free Project

Days 9–12 · Jun 22–25 · Demo Day Jun 29

Your idea, your call. The only rule: it has to be a webpage that lives on the internet.

BUILD WHATEVER YOU ACTUALLY WANT

  • A fan page for something you love — band, game, show, sport, anime
  • A tool that solves a real problem you actually have
  • A game you’d actually play
  • A portfolio that collects all three of your projects
  • A resource page for something your school is missing
  • Something completely original — whatever you genuinely care about

THE ONLY REQUIREMENTS

  • It’s a single HTML file (CSS in a <style> tag, JS in a <script> tag is fine)
  • It’s live on GitHub Pages with a working URL
  • You can do a 3-minute demo of it on Jun 29

DEMO DAY — JUN 29

3 minutes. Open your Pages URL on the projector. Show what it does. Tell us one thing that was hard to build. That’s it.
