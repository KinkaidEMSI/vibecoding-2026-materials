Get Started — Set Up Your Repo & Codespace
🚨
Critical — Do not skip the README step. When creating your repo, you must check "Add a README file." If you skip it, the repo is empty and Codespaces will not work. You will have to delete the repo and start over.
  1. 1
    Create a free GitHub account Go to github.com → Sign up. Use a personal email. Pick a username you won't be embarrassed by in 5 years.
  2. 2
    Submit the intake form Fill out the GitHub & Copilot Intake Form so your instructor can add you to the org and assign your Copilot seat.
  3. 3
    Create a new public repository Click + in GitHub top-right → New repository. Name it my-website. Set visibility to Public.
  4. 4
    Check "Add a README file" — required Before clicking Create, scroll down and check the box that says Add a README file. This is what makes Codespaces work. Do not skip this.
  5. 5
    Open a Codespace Inside your new repo: click the green Code button → Codespaces tab → Create codespace on main. Wait 60–90 seconds. VS Code opens in your browser.
  6. 6
    Create your index.html In the file explorer (left panel): right-click → New File → name it index.html. This is your website. Everything you build goes here.
  7. 7
    Preview with Live Server — not Python Right-click index.html in the file explorer → Open with Live Server. A preview opens. Every time you save (Ctrl+S), it updates automatically. Do not use the Python server.
✅
You're ready when: You can see a live preview of your page that updates every time you save. Copilot chat should be visible in the right panel. If anything is missing, tell your instructor before starting to build.

Project 1 — Personal Website
📋
Before you start building: Fill out the planning sheet your instructor gives you. Sketch your layout and plan your sections on paper first. Students who plan before they prompt build better sites faster.

Your personal website must have all 6 sections below. Your TA will check for each one. You can add more — the more original and detailed, the better your grade.

Required
Section 01

About Me

3–5 sentences about who you are. Your personality, background, what makes you you — not just your name.

Required
Section 02

Interests & Hobbies

At least 2–3 things you're into. Include images or icons. Sports, music, art, gaming, cooking — anything real.

Required
Section 03

Favorite Things

Music, movies, food, places, games — a collection of your taste. Make it feel like a playlist of who you are.

Required
Section 04

Something I'm Proud Of

One real win — a sport, a project, a performance, an achievement. Write about it like it matters.

Required
Section 05

A Goal or Dream

Where do you want to go? What do you want to build, become, or experience? Be specific — not just "be successful."

Required
Section 06

Find Me

A social handle, email, or any link you're comfortable sharing. Only share what you're okay with anyone seeing.

💡
Going beyond the minimum? Add a photo gallery, embedded playlist, dark mode toggle, animations, a quote rotator, a skills section, a timeline. Students who go beyond the requirements build the strongest portfolios.

How to Use GitHub Copilot to Build Your Site
⚡
The golden rule — always end your prompt with this:
"Give me the complete updated file in a single code block."
Without this, Copilot gives you a fragment. You paste it wrong. Your site breaks. This one line prevents 90% of errors.
❌ Too vague — don't do this make me a website
❌ Too vague — don't do this add an about section
✅ Specific — this works Build me a personal website called "Jordan's World" with a dark navy background and gold text. Include an About Me section (I play basketball and love anime), an Interests section (basketball, anime, cooking), and a Favorite Things section (Naruto, tacos, Houston Rockets). Use a clean modern sans-serif font. Give me the complete updated file in a single code block.
✅ Adding to your existing site — always paste your current code Here is my current index.html: [paste your entire current file here]. Add a Goals section after Favorite Things. My goal is to play D1 basketball and study sports medicine. Keep all existing content and styles exactly the same. Give me the complete updated file in a single code block.
✅ Fixing a visual problem Here is my current index.html: [paste your entire current file here]. The text in my About section is too small on mobile and the layout breaks on narrow screens. Fix the font sizes and make it responsive for phones. Give me the complete updated file in a single code block.

Save Your Work — Commit Every Session
⚠️
Codespaces do not auto-save to GitHub. If you close your browser without committing, your work is in a suspended container — not safe. Commit at the end of every session. Takes 30 seconds.
  1. 1
    Open the Terminal Menu → Terminal → New Terminal. A panel opens at the bottom of the screen.
  2. 2
    Stage your changes Type git add . and press Enter.
  3. 3
    Commit with a real message Type git commit -m "add about section and interests" — write what you actually did, not "update."
  4. 4
    Push to GitHub Type git push and press Enter. Your work is now saved on GitHub and will not be lost.
🏆
Good commit messages: "add interests section with images" · "fix mobile layout" · "add goals and social links" · "change to dark mode colors"
Bad: "stuff" · "update" · "idk" · "aaaaaa"
