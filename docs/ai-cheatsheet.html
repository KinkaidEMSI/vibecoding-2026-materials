AI Cheat Sheet

Prompt patterns that work. How to get complete, usable code every time — and what to do when you don't.

🔑 The One Rule That Prevents 90% of Problems

Every time you ask Gemini (or any AI) to write or change code, end your prompt with this exact phrase:

Give me the complete updated file in a single code block.

Never paste partial code into the middle of a file. Always replace the whole thing. Partial pastes cause cascading bugs that are hard to track down.

Core Prompt Patterns
🏗️ Start from Scratch
Use this when:

Building a new section, page, or feature from zero.

Create a single HTML file for a personal website. It should have: - A header with my name "[Name]" in large text - A short bio paragraph (2–3 sentences) - A section showing 3 of my interests as cards - A simple dark navy and white color scheme Give me the complete file in a single code block.
✏️ Modify Existing Code
Use this when:

Changing something that already exists. Always paste your current code.

Here is my current index.html: [paste your entire current file here] Please change the background color to a dark navy blue and make the heading font larger. Do not change anything else. Give me the complete updated file in a single code block.
🐛 Debug a Problem
Use this when:

Something looks wrong or broken in the browser.

Here is my HTML file: [paste your current file here] The problem I see is: my navigation links don't show up on mobile — the menu just disappears. What is causing this and how do I fix it? Give me the complete corrected file in a single code block.
✨ Add a Feature
Use this when:

Your base page works and you want to add something new.

Here is my current index.html: [paste your current file here] Add a "dark mode" toggle button in the top right corner. When clicked, it should switch the background to dark navy and text to white. Clicking again switches back. Use only HTML, CSS, and JavaScript — no libraries. Give me the complete updated file in a single code block.
Prompts for the Interactive Project (Phase 2)
Starting a quiz
Create a single HTML file for a multiple-choice quiz about [topic]. Include 5 questions, each with 4 answer options. After each answer, show if it was correct or incorrect. Show the final score at the end with a "Play Again" button. Use a clean, friendly design — nothing complicated. Give me the complete file in a single code block.
Starting a clicker game
Create a single HTML file for a clicker game. Clicking a big button earns 1 point. Show a running score counter at the top. Every 10 clicks, unlock a "power-up" that makes each click worth 2 points. Keep the design simple and fun. Give me the complete file in a single code block.
Starting a random generator
Create a single HTML file for a random [thing] generator. It should have a big "Generate" button in the center. Clicking it picks a random [thing] from a list of at least 20 options and displays it in large text with a fun animation. Give me the complete file in a single code block.
Prompt Do's and Don'ts
✓ DO — Good Prompting
  • Be specific about what you want to see
  • Describe the outcome, not the code technique
  • Always paste your current file when modifying
  • Always end with "complete file in a single code block"
  • Tell it what NOT to change if you want to keep something
  • Ask one thing at a time if the request is complex
  • If it's wrong, describe exactly what's still broken
✗ DON'T — Weak Prompting
  • Don't say "make it look better" — better how?
  • Don't say "add some cool stuff" — what stuff?
  • Don't paste partial code and ask it to fill gaps
  • Don't say "fix my code" without saying what's broken
  • Don't accept code you don't understand — ask why
  • Don't give up after one try — iterate and be specific

When AI Gives You Bad Code
🔄 The Iteration Loop
  • Paste it, test it. Does it do what you asked? If yes, great. If no, keep going.
  • Describe the problem exactly. Not "it doesn't work" — "the button exists but clicking it does nothing."
  • Paste the broken code back. AI has no memory. It can't fix what it can't see.
  • Try a different angle. Sometimes rephrasing the question gets a better result.
  • Ask for an explanation. "Why did you do it that way?" teaches you more than just accepting the answer.
