Day 1 · June 8, 2026 · 80 Minutes

Setup & Onboarding

By the end of today you will have a live website on the internet. Zero experience required. Follow every step in order.

🎯 Today's Goal
  • GitHub account created and joined to the class org
  • GitHub Classroom assignment accepted — your repo exists
  • Codespace open in your browser — VS Code is live
  • Gemini AI is active and responding
  • Your index.html is visible in your browser via Live Server
Part 1 — GitHub Account (10 min)
1
Create a free GitHub account
~5 min

If you already have a GitHub account, skip to step 2.

  1. 1Go to github.com and click Sign up
  2. 2Enter your email address. Use a personal email — not your school email.
  3. 3Create a username. Keep it professional — you may use this for years. Example: jsmith2026 or janedoe
  4. 4Verify your email address when GitHub sends you a confirmation link.

Username tip: No spaces, keep it lowercase, avoid numbers at the start. This appears in your project URLs.

2
Accept the class organization invite
~2 min

Your instructor will share a link. Click it while logged into GitHub. This adds you to the KinkaidEMSI organization.

⚠️ Must be logged in to GitHub when you click the link

If you click the link while logged out, it won't attach to your account. Log in first, then open the link.

Part 2 — Accept Your Assignment (10 min)
3
Accept the GitHub Classroom assignment
~3 min

Open the GitHub Classroom invitation link below while logged into GitHub.

  1. 1Open this link: https://classroom.github.com/a/xuJPS0u1
  2. 2Click Accept this assignment
  3. 3Wait ~15 seconds. GitHub creates your personal repo. The URL will look like:
    github.com/KinkaidEMSI/vibeproject1-2026-yourusername
  4. 4Click the link GitHub gives you to open your repo.
Part 3 — Open Your Codespace (15 min)
4
Launch your Codespace
~10 min (first time is slow)

A Codespace is VS Code running in your browser. No downloads. No installs. It lives in the cloud.

  1. 1In your repo, click the green Code button
  2. 2Click the Codespaces tab
  3. 3Click Create codespace on main
  4. 4A new browser tab opens. VS Code loads. This takes 1–2 minutes the first time. Normal. Wait.
⚠️ First launch takes 60–90 seconds — this is normal

The container is being built. Don't close the tab or click anything. When VS Code appears, you're in.

5
Verify Gemini AI is working
~3 min

Gemini should be installed automatically. Let's confirm.

  1. 1Open the terminal in VS Code: top menu → Terminal → New Terminal
  2. 2Type this and press Enter:
echo $GEMINI_API_KEY

You should see a long string of characters. If you see something, Gemini is ready. If it's blank, tell your instructor.

  1. 3Look for the Gemini icon in the left sidebar. Click it to open the chat panel.
  2. 4Type "Hello" and press Enter. If Gemini responds, you're good to go.
Part 4 — Your First HTML File (25 min)
6
Open index.html and launch Live Server
~5 min

Your repo already has an index.html file. Let's open it and see it in a browser.

  1. 1In the left file panel, click index.html to open it
  2. 2Right-click anywhere in the file → click Open with Live Server
  3. 3A preview opens. This is your site. Every time you save the file, it refreshes automatically.

Keyboard shortcut: Ctrl + S (Windows) or Cmd + S (Mac) to save. Get used to saving constantly.

7
Ask Gemini to add your name to the page
~10 min

This is vibe coding. You're not typing HTML — you're directing AI to write it. Try this prompt in the Gemini chat panel:

Add an h1 heading with my name "[Your Name]" and a short paragraph that says I'm a student in the EMSI summer program at Kinkaid School. Give me the complete updated index.html file in a single code block.
  1. 1Copy the entire code block Gemini gives you
  2. 2Select all the text in index.html: Ctrl+A (or Cmd+A)
  3. 3Paste the new code: Ctrl+V (or Cmd+V)
  4. 4Save: Ctrl+S. Check the Live Server preview — your name should appear.

Key rule: Always ask Gemini for "the complete updated file in a single code block." Never paste partial code — it breaks things.

Part 5 — Your First Commit (15 min)
8
Add, commit, and push
~10 min

A commit is a saved snapshot. Pushing sends it to GitHub. This is the loop you'll do every session.

  1. 1Open the terminal (Terminal → New Terminal if it's closed)
  2. 2Type these three commands, pressing Enter after each:
git add . git commit -m "Day 1: added my name to index.html" git push
  1. 3Go to github.com/KinkaidEMSI/vibeproject1-2026-yourusername — your commit should appear.
⚠️ Commit messages matter — be specific

Bad: "update". Good: "Day 1: added heading and intro paragraph". Your instructor reads these.

9
Enable GitHub Pages — get your live URL
~5 min

GitHub Pages turns your repo into a public website. One-time setup.

  1. 1In your repo on GitHub, click Settings
  2. 2In the left menu, click Pages
  3. 3Under "Source", select Deploy from a branch
  4. 4Set branch to main, folder to / (root)
  5. 5Click Save. Wait ~1 minute. Refresh the Settings → Pages page.
  6. 6Your live URL appears: kinkaidemsi.github.io/vibeproject1-2026-yourusername

That's your project URL. Every push to main automatically updates it. This is what you'll submit to Google Classroom.
