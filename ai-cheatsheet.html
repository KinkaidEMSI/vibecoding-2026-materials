Google AI Studio — Coding Cheat Sheet

Your AI assistant aistudio.google.com/prompts/new_chat Sign in with your Google account · Free · No card needed
Step 1 — Settings to use every time
  • Model Gemini 3 Flash
  • Thinking Minimal
  • Grounding / Search OFF
  • Temperature Default (1.0)
  • System instructions Leave blank
Why Gemini 3 Flash? It's the fastest free model that's actually good at writing code. Gemini 3.1 Flash-Lite is faster but weaker — use it only when 3 Flash is slow.
Step 2 — How to talk to it (the formula)
[What you want] + [specific details] + [constraints]
AI is not a search engine. You're not typing keywords — you're giving instructions to a coding assistant that will do exactly what you say. The more specific you are, the better the code you get back.

Bad prompt: make my site look cool
Good prompt: Add a dark navy background to the body, change the h1 font color to bright orange (#ff6b35), and add a 2px white bottom border to the nav links on hover. Only change the CSS, don't touch the HTML.
Starter prompts — use these to begin
  • Build from scratch
    Write a single HTML file for a personal website about [your topic]. Include: a header with my name, a short about section, and one section about [hobby/interest]. Use CSS inside a <style> tag. Make it dark themed with orange accents. No JavaScript needed.
  • Build an interactive project
    Write a single HTML file for a [quiz / clicker game / random generator / trivia game] about [topic]. All CSS and JavaScript in the same file. Keep it simple — one screen, no routing, no external libraries.
Iteration prompts — after you have code running
  • Add a feature
    Here is my current HTML file: [paste code]. Add a [feature — e.g. "score counter that increments when the button is clicked"]. Don't change anything else. Return the full updated file.
  • Change the style
    Here is my CSS: [paste]. Change only the colors — replace all instances of blue with [color]. Keep everything else identical.
  • Make it mobile-friendly
    Here is my code: [paste]. Add a media query so on screens narrower than 600px the layout stacks vertically instead of side by side. Return only the CSS changes, not the full file.
Debug prompts — when something breaks
  • Something is broken
    Here is my HTML: [paste code]. When I [describe the problem — e.g. "click the button"], nothing happens. What is wrong? Explain what's broken, then give me the fixed code.
  • It doesn't look right
    Here is my CSS: [paste]. My [element name — e.g. "nav bar"] is not centered on the page. Fix just the CSS for that element. Show me the corrected lines only.
  • I got an error message
    I see this error in the browser console: "[paste error]". Here is the JavaScript that's causing it: [paste code]. What does this error mean and how do I fix it?
Good prompting habits
  • Always paste your current code when asking for changes — AI can't see your screen
  • Say "return the full updated file" or "return only the changed lines" — pick one
  • Describe what you want to happen, not just what's wrong
  • If the first answer is bad, add more detail and ask again — don't give up
  • Don't type just "fix it" or "make it better" — AI needs specifics
  • Don't paste 500 lines of code without explaining what part to focus on
  • Don't trust AI blindly — read the code before you paste it into your file
Plan B
If AI Studio is down → go to gemini.google.com. Same Gemini model, same Google account. Slightly simpler interface. Your work is identical.
When things go wrong
  • AI gave me code that doesn't work Paste the broken code back in and say "this doesn't work — [explain what happens]. Fix it."
  • AI changed something I didn't ask it to change Say "undo that change and only do what I asked." Or use Git — the last commit has your old code.
  • AI keeps giving me the wrong answer Start a new chat. Long conversations sometimes confuse the model. Paste your full code fresh.
  • I don't understand what the code does Ask it: "Explain what this code does line by line in simple terms." Understanding it = better prompts later.
  • "Rate limit" or "too many requests" error Wait 60 seconds and try again. You have 1,500 requests per day — don't spam it.
The commit loop — don't forget this
Every time you get code working in your browser, commit it. If you break something later, your last commit is your save point.
  • 1. git add . stage everything
  • 2. git commit -m "what you did" save snapshot
  • 3. git push go live on Pages
Commit message tips:
✓ "added score counter to quiz"
✓ "changed background to dark mode"
✗ "fixed stuff" — too vague, useless in a week
